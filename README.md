# Senior QA Automation Assignment — UI (Playwright + TypeScript)

Automates the **Demo Web Shop** UI flow:

- Website: https://demowebshop.tricentis.com
- Test case: **Place order with multiple products** with **price calculation checks**

## Tech

- Playwright Test + TypeScript (Page Object Model)
- External test data: JSON
- Reporting: Playwright HTML report + JUnit XML

## Prerequisites

- Node.js 18+ (Node 20/22 recommended)

## Setup

```bash
npm install
npx playwright install
```

## Environment variables (no secrets committed)

This project **does not commit** any `.env` file. Create one locally:

```bash
copy .env.example .env
```

Supported variables:

- `BASE_URL` (optional) — defaults to `https://demowebshop.tricentis.com`
- `HEADED` (optional) — set `1` to run headed (non-headless)
- `USER_EMAIL` / `USER_PASSWORD` (optional) — reserved if you want to extend tests to authenticated checkout

Notes:

- The provided UI test runs **guest checkout** and generates a **unique email per run**, so it does not require personal accounts.

## Running tests

```bash
npm test
```

Run with UI runner:

```bash
npm run test:ui
```

## Reports

- HTML report: `playwright-report/` (open via `npm run test:report`)
- JUnit XML: `test-results/junit.xml`

## Project structure (POM)

- `src/pages/` — Page Objects
- `src/data/orderData.json` — external test data (products, billing address template, etc.)
- `src/utils/` — shared helpers (money parsing, JSON loader, unique email)
- `tests/` — Playwright test specs

## API Automation (Postman)

The Postman collection export for the Swagger Petstore APIs is included here:

- [postman/petstore.collection.json](postman/petstore.collection.json)

Covered scenarios:

- `POST /pet` valid create (stores `petId` for follow-up requests)
- `GET /pet/{petId}` fetch created pet (200)
- `POST /pet` invalid input (expects 400/405)
- `GET /pet/{petId}` invalid id supplied (400)
- `GET /pet/{petId}` pet not found (404)

How to run:

1. Import the collection JSON into Postman.
2. Collection variables include `baseUrl` (defaults to `https://petstore.swagger.io/v2`).
3. Run the collection via Collection Runner to execute all tests.
